Options: - get: Appends form data to the URL (visible). - post: Sends form data in the HTTP body (secure for sensitive data). enctype Specifies the encoding type for form data. Used with method="post". Options: - application/x-www-form-urlencoded (default). - multipart/form-data (for file uploads). - text/plain (plain text). target Specifies where to display the response after form submission. Options: - _self (default): Same window/tab. - _blank: New tab/window. - _parent: Parent frame. - _top: Full body of the window. autocomplete Enables or disables auto-completion for form fields. Options: - on (default): Enables auto-completion. - off: Disables auto-completion. novalidate Disables form validation on submission. name Assigns a name to the form for identification (e.g., in JavaScript). rel Specifies the relationship between the form and its target (used with target). 2. Element Attributes Attribute Description type Specifies the type of input. Examples: text, password, email, number, file, etc. name Assigns a name to the input field for form submission and identification. value Specifies the default value of the input field. placeholder Provides a short hint about the expected value (displayed inside the field). required Marks the field as mandatory. maxlength Specifies the maximum number of characters allowed. minlength Specifies the minimum number of characters required. pattern Validates the input against a regular expression. readonly Makes the input field read-only (non-editable). disabled Disables the input field (grayed out and non-editable). autofocus Automatically focuses the input field when the page loads. multiple Allows multiple values (e.g., for file or email inputs). step Specifies the interval for numerical input (e.g., step="2" for type="number"). min and max Define minimum and maximum values for numerical input. size Specifies the visible width of the input field (in characters). list Links the input field to a for predefined suggestions. 3.
